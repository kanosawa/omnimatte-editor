# 仕様書インデックス

このディレクトリは Omnimatte Editor の仕様書一式です。Claude Code が実装する際の単一の参照源（Single Source of Truth）として機能します。

## 読む順序

実装着手時は以下の順に読むことを推奨します。

| # | ファイル | 概要 |
|---|---|---|
| 1 | [01-overview.md](01-overview.md) | プロジェクトの目的・機能要件・技術スタック |
| 2 | [02-architecture.md](02-architecture.md) | 全体アーキテクチャとディレクトリ構成 |
| 3 | [03-backend.md](03-backend.md) | バックエンド（FastAPI + SAM2）仕様 |
| 4 | [04-api.md](04-api.md) | フロントエンド⇔バックエンド間のAPI契約 |
| 5 | [05-frontend-structure.md](05-frontend-structure.md) | フロントエンドの全体構造（コンポーネント階層） |
| 6 | [06-ui-layout.md](06-ui-layout.md) | UI 3段構成の詳細レイアウト |
| 7 | [07-pixi-canvas.md](07-pixi-canvas.md) | Pixiキャンバスクラスの設計 |
| 8 | [08-state-management.md](08-state-management.md) | zustandによる状態管理と VideoElement 同期 |
| 9 | [09-state-transitions.md](09-state-transitions.md) | バウンディングボックスとSAM2ボタンの状態遷移 |

## レイヤ別の読み順

特定の作業を始めたいときは、以下の最小セットを読めば着手可能です。

- **バックエンド作業**: 01 → 02 → 03 → 04
- **フロントエンド作業（UI）**: 01 → 02 → 05 → 06 → 09
- **フロントエンド作業（描画）**: 01 → 05 → 07 → 08
- **状態設計の修正**: 08 → 09

## 用語集

| 用語 | 意味 |
|---|---|
| **BBox** | バウンディングボックス。ユーザーがキャンバス上でドラッグ指定する矩形領域 |
| **合成 mp4** | バックエンドが `/segment` のレスポンスとして返す mp4。原動画にマスク領域を半透明＋着色でアルファブレンド済み |
| **重畳表示** | オリジナル動画の上に、半透明＋着色したマスクを重ねて表示すること。本プロジェクトではバックエンド側で合成済み mp4 として行う |
| **VideoElement** | HTML5 の `<video>` 要素。フロントエンドでmp4をデコード/再生する主体 |
| **Pixi** | PixiJS。WebGLベースの2D描画ライブラリ |
| **セッション** | バックエンド側で保持される動画。常に最大1件のみ保持し、新規作成時に直前のセッションは自動的に破棄される |
| **ローカル運用** | ユーザーのマシン上で FastAPI サーバ（`127.0.0.1:8000`）を直接起動して使う形態 |
| **GPU サーバ運用** | クラウドの GPU 機にバックエンドを置き、クライアント側で SSH ポート転送経由（`localhost:8000` → サーバ `127.0.0.1:8000`）で接続する形態 |

## ドキュメンテーション規約

- 言語: 日本語
- 図: Mermaid を使用
- 各仕様書末尾に **実装チェックリスト** を記載（Claude Code がタスク化しやすい粒度）
- ファイル名は `NN-name.md` 形式（NN は2桁、追加余地のため番号は飛ばしても可）
- 仕様変更時は該当ファイルを更新し、関連する他ファイルへのリンクも見直す

## 範囲外（Out of Scope）

以下は本仕様の対象外です。実装時は最小限の対応のみ行い、本格対応は別途仕様化します。

- アプリ層の認証・認可（アクセス制御はサーバ側のネットワーク制限／SSH に委ねる）
- 複数物体の同時セグメンテーション（仕様上、1物体のみ）
- マスクの編集・補正UI
- 動画エクスポート（合成結果の書き出し）
- 多言語対応（UIは日本語/英語のいずれかに固定）
